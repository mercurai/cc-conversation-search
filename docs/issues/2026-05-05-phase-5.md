# Phase 5: define the harness-facing JSON output contract

## Objective

Expose mined transcript facts in a stable machine-readable shape so downstream harness and autoresearch systems can consume this repo without scraping human text output.

## Relevant files

- `src/conversation_search/core/session_miner.py`
- `src/conversation_search/cli.py`
- `README.md`
- optional dedicated contract doc if needed

## Required changes

1. Define the JSON schema emitted by `mine-session --json`.
2. Include at least:
   - resolution metadata
   - transcript path
   - time range
   - record counts
   - tool counts
   - shell commands
   - files touched
   - extracted errors
   - local DB correlation summaries
3. Document the intended downstream usage.
4. Keep this repo read-only with respect to downstream autoresearch databases for now.

## Acceptance criteria

- an external consumer can parse the output without brittle text scraping
- the schema is documented in the repo
- the CLI output remains stable across repeated runs on the same transcript
