## Objective

Add regression tests for the exact resolution and transcript-mining edge cases observed in real Claude Code sessions.

## Relevant files

- `tests/test_session_miner.py`
- new fixture files under `tests/fixtures/` as needed

## Required changes

1. Add fixtures for:
   - explicit Windows transcript path input
   - `tree` returning JSON with `"error"`
   - Codex-side filename or session mentions that must not resolve
   - hook error attachments
   - mixed real-world record types seen in Claude transcripts
2. Add assertions for:
   - resolved transcript path
   - preserved project path semantics
   - shell command extraction
   - file-touch extraction
   - attachment-derived error extraction
3. Keep tests fast and self-contained.

## Acceptance criteria

- the miner test suite covers resolution, parsing, and failure extraction behavior
- the suite fails if Codex incidental mentions start resolving again
- the suite fails if `tree` JSON errors are treated as success
