# Implement the shared session-mining core and packaging hardening

## Goal

Turn `mercurai/cc-conversation-search` into the durable shared transcript-resolution and session-mining layer for:

- direct CLI usage
- Claude Code packaging
- Codex packaging
- downstream harness and autoresearch consumers

This repo should own the canonical logic. Packaging layers should stay thin.

## Reference plan

- `docs/plans/2026-05-05-implementation-plan.md`

## Why this work exists

Recent end-to-end testing confirmed the CLI can now resolve and mine a real Claude Code transcript, but the implementation still has important seams and correctness gaps:

- project-path normalization is degraded in resolved output
- session resolution rules need clearer boundaries between true transcript resolution and secondary evidence
- text-only report output is not a durable downstream contract
- local install/repair flows were brittle under stale launcher conditions
- regression coverage is not yet broad enough for real Claude transcript shapes

## Constraints

- Keep transcript logic in this repo, not duplicated in `claude-code-config`
- Preserve existing CLI commands
- Treat `tree` JSON responses containing `"error"` as unresolved even when exit code is `0`
- Do not treat incidental Codex session mentions as proof of Claude transcript resolution
- Keep downstream autoresearch DB writes out of this repo for now

## Exit criteria

- the CLI resolves and mines real transcripts reliably on mixed Windows/Linux setups
- Codex and Claude adapters point at the same core behavior
- the package exposes a stable machine-readable contract for downstream ingestion
- install and recovery steps are documented and tested
- regression tests cover the failure modes seen in real session transcripts
