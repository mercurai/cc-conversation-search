# Phase 2: add a structured miner contract and align the Claude/Codex adapters

## Objective

Promote `mine-session` from a text-only report generator into a reusable structured contract that both packaging layers consume.

## Relevant files

- `src/conversation_search/core/session_miner.py`
- `src/conversation_search/cli.py`
- `codex-skills/claude-session-miner/SKILL.md`
- `codex-skills/claude-session-miner/scripts/mine_claude_session.py`
- `skills/conversation-search/SKILL.md`
- `README.md`

## Required changes

1. Define a stable programmatic API in `session_miner.py`.
2. Add a machine-readable output mode for `mine-session`, preferably `--json`.
3. Keep the current text report, but render it from structured data instead of assembling the report as the only output.
4. Align the Codex and Claude skills so both state the same behavioral rules:
   - use `tree` for session IDs
   - explicit transcript path is the preferred fallback
   - incidental Codex mentions are not resolution
5. Remove packaging-layer logic that duplicates behavior already implemented in the core module.

## Acceptance criteria

- `mine-session --json` returns stable structured data for the same real transcript used in Phase 1
- human-readable output still works
- Codex and Claude skill docs no longer drift on resolution semantics
- packaging adapters are thin wrappers around the shared core behavior
