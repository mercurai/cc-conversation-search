# Phase 3: harden install and packaging recovery

## Objective

Make local installs and recovery from stale launchers deterministic across the supported packaging surfaces.

## Relevant files

- `install.sh`
- `README.md`
- `.codex-plugin/plugin.json`
- `.claude-plugin/README.md`
- `.claude-plugin/INSTALL.md`

## Required changes

1. Harden `install.sh` for repo-path installs using:
   - `uv tool install /path/to/repo`
2. Detect and document stale launcher recovery, especially when `.local/bin/cc-conversation-search.exe` exists without a healthy UV tool environment.
3. Clarify the difference between:
   - repo checkout
   - installed CLI
   - plugin registration
4. Ensure the manifest version, package version, and docs all agree.
5. Document a known-good recovery path for Windows Git Bash and Unix shells.

## Acceptance criteria

- a stale or orphaned local launcher can be repaired with documented steps
- the local repo install path produces a healthy CLI
- docs describe how Claude Code plugin packaging and Codex packaging relate to the same core package

## Verification

```bash
bash install.sh
cc-conversation-search --version
cc-conversation-search --help
uv tool list
```

## Notes for implementation

- The previous failure mode was a half-removed `uv` launcher that still existed on `PATH`.
- Do not assume the presence of a healthy UV tool environment just because the executable name exists.
