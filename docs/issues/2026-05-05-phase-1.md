# Phase 1: harden session resolution and path normalization

## Objective

Make session resolution deterministic and correct across:

- indexed `tree` lookup
- explicit transcript path input
- raw `~/.claude/projects/**/<session>.jsonl` fallback
- Codex-side filename matches treated only as secondary evidence

## Relevant files

- `src/conversation_search/core/session_miner.py`
- `src/conversation_search/cli.py`
- `tests/test_session_miner.py`

## Required changes

1. Split resolution into explicit ordered stages:
   - `cc-conversation-search tree <session-id> --json`
   - explicit transcript path
   - raw Claude transcript search
   - Codex-side filename matches recorded but non-resolving
2. Preserve and normalize Windows transcript paths correctly.
3. Fix project-path degradation such as `D//projects/claude/code/config`.
4. Keep `tree` parsing resilient when the returned JSON shape varies slightly.
5. Return stable structured resolution metadata from the core miner.

## Acceptance criteria

- `tree` results with `"error"` are treated as unresolved even if exit code is `0`
- explicit `--transcript` paths resolve correctly on Windows
- incidental Codex session mentions are never treated as resolution
- the mined output preserves a correct project path representation
- the target transcript below resolves correctly:
  - `C:\Users\svere\.claude\projects\D--projects-claude-code-config\5f21428b-056a-473c-a6f8-c2ab49aa90bb.jsonl`

## Notes for implementation

- Prefer additive changes over DB-destructive behavior.
- If indexed metadata needs compatibility cleanup, handle it in code rather than forcing a rebuild.
